# LTS Agent v0.7.1 acceptance

Version 0.7.1 was tested on 2026-07-20 using a disposable clone of LBI 1.0
Build 002 template 901. This patch release changes only timer re-arming and the
reported semantic version; inventory, configuration, and Core contracts remain
unchanged from v0.7.0.

- Package: `lts-agent_0.7.1_amd64.deb`
- Package SHA-256:
  `c820673997d4f21492d753f11000af20e67034357db98d81e8511ecab66f56d5`
- Target: Ubuntu 24.04.4 LTS, amd64, LBI 1.0 Build 002

Formatting, vet, normal tests, race-enabled tests, native build, static
Linux/amd64 build, package staging, Debian metadata, installed paths, and file
modes passed. The installed timer contains `OnBootSec=2min`,
`OnActiveSec=2min`, `OnUnitActiveSec=5min`, `RandomizedDelaySec=30s`, and
`Persistent=true`.

The live regression test installed v0.7.0, ran the service, removed and purged
the package, and installed v0.7.1 while retaining the prior systemd timer
history. No manual service start was performed after v0.7.1 installation. The
timer immediately entered `active (waiting)` with a future activation-relative
deadline instead of the v0.7.0 `active (elapsed)` state.

Automatic runs started at 13:41:55 and 13:47:35 CAT. Both ran as
`lts-agent:lts-agent`, exited 0, reported validation `passed`, health `healthy`,
and Core disabled, and finished before the next run began. A subsequent trigger
remained scheduled, proving that the activation-relative recovery transitions
into the normal five-minute cadence without overlap.
